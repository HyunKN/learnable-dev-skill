# Core Principles

## Good AI Use Is Not About Maximum Automation

The user is using AI well when they still control these:

- what is being built
- why it is being built that way
- how important decisions are documented
- how the implementation is validated

## Healthy Concern Is a Good Sign

Worrying about direction, technical choices, security, maintainability, and whether a project is becoming a fragile sandcastle is not a bad sign. It usually means the user is ready for a more durable engineering workflow.

## Durable Development Loop

Prefer this loop:

1. decide
2. implement
3. validate
4. document

## Prefer Explainable Code Over Merely Impressive Code

The goal is not just code that works. The goal is code and decisions that can be explained later.

## Human and AI Roles

Human:

- direction
- goals
- prioritization
- understanding
- final judgment

AI:

- implementation support
- alternatives
- review
- documentation help
- explanation

## Learning While Building

The user should leave behind more than finished features. They should also leave behind:

- technical understanding
- short notes
- explanations of why a technology was chosen
- known tradeoffs
- possible improvements
