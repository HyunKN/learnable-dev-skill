# Guide - Chat Insight Capture

Do not let meaningful chat insights disappear as one-off conversation. Capture them as project knowledge when they are likely to help again.

## When To Capture

Capture an insight when one of these becomes clearer during the conversation:

- a concept becomes easier to explain
- a role, boundary, or ownership line becomes clearer
- the reason behind a refactoring direction becomes clearer
- an explanation appears that will likely help again
- a reusable working pattern or decision criterion emerges

## How To Capture

- Update an existing document when the insight clearly belongs there.
- Create a new note when no existing document is a good fit.
- If follow-up work depends on the insight, make it visible in handoff context as well.

## Minimum Fields

- Insight:
- Why it matters:
- Where it applies:
- Conversation or task context:
- Follow-up, if needed:

## Suggested Filenames

For reusable reference docs:

- `guide-...`
- `playbook-...`
- `template-...`

For project-specific insight notes:

- `insight-YYYY-MM-DD-what-became-clear.md`

Examples:

- `insight-2026-03-30-document-title-clarity.md`
- `insight-2026-03-30-when-to-update-an-existing-doc.md`
