# Learning Playbook

## Goal

Use AI in a way that leaves behind technical understanding, not just finished code.

## Best Learning Pattern

For each meaningful feature, keep track of:

- why the feature exists
- which files changed
- why the chosen technology was used
- what alternatives were rejected and why

## Practical Loop

1. State your own structure first.
2. Let AI improve it.
3. Re-explain the result after implementation.
4. Dive deeper only where understanding is weak.

## Coding Skill Habits

- Try small functions or UI events yourself.
- Read AI-changed files at least once.
- Summarize unfamiliar APIs or syntax.
- Explain why a piece of code works in one paragraph.

If you cannot explain it, it is not fully yours yet.

## Five Questions Per Technology

For any important tool or library, be able to answer:

1. What is it?
2. Why was it used?
3. What are its strengths?
4. What are its weaknesses?
5. How is it used in this project?
